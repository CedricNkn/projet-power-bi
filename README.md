# projet-power-bi
<p><img src="https://user.oc-static.com/upload/2023/05/16/16842499242082_OC-Bannieres-Projet_Student-Scenario_Scenario.png" alt="Scénario"></p>
<p>Vous avez la possibilité de réaliser un tableau de bord portant sur des&nbsp;<strong>données différentes</strong>&nbsp;de celles présentées dans le projet.&nbsp;</p>
<p>Prenez toutefois le temps de lire l’option A afin de vous inspirer le plus possible du contexte et du cadre de ce projet.</p>
<p>Concernant les données, choisissez un sujet qui vous intéresse, qui vous parle ou qui est lié à un secteur dans lequel vous aimeriez travailler. Vous devez faire valider le choix de dataset par votre mentor. Discutez des critères d'éligibilité de votre dataset, notamment sa taille, avec votre mentor.</p>
<aside data-claire-semantic="information">
<p>Voici quelques pistes de sources de données :&nbsp;</p>
<ul>
<li>plateforme opendata du gouvernement francais :&nbsp;&nbsp;<a class="custom-link" href="https://www.data.gouv.fr/fr/">data.gouv</a></li>
<li>plateforme opendata de la marie de Paris :&nbsp;<a class="custom-link" href="https://opendata.paris.fr/pages/home/">opendata.paris</a>&nbsp;</li>
<li>site de l’INSEE :&nbsp;<a class="custom-link" href="https://www.insee.fr/fr/statistiques?categorie=1">insee.fr</a></li>
<li>site de la banque mondiale :&nbsp;<a class="custom-link" href="https://datacatalog.worldbank.org/home">datacatalog.worldbank</a></li>
<li>site de l’OCDE :&nbsp;<a class="custom-link" href="https://data.oecd.org/fr/">data.ocde&nbsp;</a></li>
<li>base de données de films :&nbsp;<a class="custom-link" href="https://www.imdb.com/interfaces/">imdb</a></li>
<li>une liste de données mises en avant par Tableau :&nbsp;<a class="custom-link" href="https://public.tableau.com/app/resources/sample-data">data</a></li>
</ul>
<p>Vous pouvez également utiliser des données que vous connaissez déjà bien : les données des&nbsp;<strong>projets 3, 4 ou 5</strong>&nbsp;notamment.</p>
</aside>
<p>Adoptez toutefois une démarche&nbsp;<strong>professionnelle</strong>,&nbsp;<strong>structurée</strong>&nbsp;et&nbsp;<strong>rigoureuse</strong>. Vous devrez :</p>
<ul>
<li>définir clairement un sujet et une problématique.</li>
<li>cadrer les différentes visualisations que vous souhaitez proposer, leurs intérêts respectifs et le niveau d’information qu’elles permettront de lire.&nbsp;</li>
</ul>
<aside data-claire-semantic="information">
<p>Il peut être facile de se perdre dans les données si elles sont trop volumineuses, trop nombreuses ou trop diverses. N'hésitez donc pas à choisir un scope précis et restreint, quitte à l’élargir par la suite.</p>
</aside>
<p>Dans certains cas, une étape de prise en main, de nettoyage, de compilation des données sera nécessaire. Vous pourrez l’effectuer avec Python, KNIME, ou directement avec Tableau, PowerBI.&nbsp;</p>
<li>Une présentation (Powerpoint ou équivalent) de votre projet. Cette présentation contiendra :&nbsp;</li>
<ul>
<li>des éléments sur le contexte, notamment des extraits de votre blueprint,&nbsp;</li>
<li>le prétraitement éventuel des données,&nbsp;</li>
<li>la pertinence de l’outil de visualisation retenu etc.</li>
<li>La version PowerBI de votre projet (fichier .pbix) ;</li>
</ul>
</ol>
